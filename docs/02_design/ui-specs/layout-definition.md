# レイアウト要件定義書：メディアDevダッシュボード

本文書は、「メディアDevダッシュボード」の全体的な画面レイアウトの構成と要件を定義するものである。

---

## 1. 基本構成

アプリケーションの基本骨格は、**画面全体の高さを占める固定サイドバー**と、その右側に配置される**ヘッダー＋コンテンツエリア**で構成される。

-   **サイドバー**: 画面左端に、上下いっぱいの高さで常に固定される。
-   **メインエリア**: サイドバーの右側全域を占める。このエリアの内部が、上下に「ヘッダー」と「コンテンツエリア」に分かれる。
-   **ヘッダー**: メインエリアの最上部に配置される。
-   **コンテンツエリア**: ヘッダーの下に配置され、このエリアのみがスクロール対象となる。

**レイアウトイメージ:**
+----------+---------------------------------------------+
|          | ヘッダー (ページタイトル | ユーザー設定)          |
|          +---------------------------------------------+
|          |                                             |
| サイドバー | コンテンツエリア                              |
| (画面縦長) | (この中がスクロールする)                       |
|          |                                             |
|          |                                             |
|          |                                             |
|          |                                             |
|          |                                             |
+----------+---------------------------------------------+

---

## 2. ヘッダーの要件

ヘッダーには、以下の要素を配置する。

-   **ページタイトル表示**: ユーザーが現在どのページを閲覧しているかを明確に示す。パンくずリスト形式で表示することも検討する。（例：「ダッシュボード / 全体」）
-   **ユーザー設定メニュー**: ログインしているユーザーのアイコンや名前を表示し、クリックするとプロファイル設定やログアウトなどのメニューが開く。

---

## 3. サイドバーの要件

サイドバーには、主要ページへのナビゲーションメニューを配置する。メニューは展開・折りたたみ可能な階層構造とする。

-   **ダッシュボード**
    -   **全体**: 全メディアを横断したサマリーダッシュボードへ遷移する。
    -   **各種メディア**: クリックすると、管理対象のメディア一覧が展開され、各メディア個別のダッシュボードへ遷移するリンクが表示される。（このリストは動的に生成される）
-   **テーブル** (予実管理ページ)
    -   **全体**: 全メディアを横断した予実管理テーブルへ遷移する。
    -   **各種メディア**: クリックすると、管理対象のメディア一覧が展開され、各メディア個別の予実管理テーブルへ遷移するリンクが表示される。

---

## 4. コンテンツエリアのレイアウト方針

コンテンツエリアのレイアウトは、ページの種類によって切り替える。

### 4.1. テーブルページ / 設定ページ
-   **レイアウト**: **1分割レイアウト**
-   **目的**: 横長のテーブルや、縦に並ぶ設定項目など、単一の要素を画面幅いっぱいに使って表示するため。

### 4.2. ダッシュボードページ
-   **レイアウト**: **縦2分割レイアウト**
-   **目的**: 重要なKPIのサマリーと、詳細なグラフ分析を同一画面で提供するため。

#### **全体構造：縦2分割**

-   **上段：カードエリア**
    -   **役割**: 最も重要なKPI（売上、費用、利益など）の数値を、カード形式で常時表示する。ユーザーはここを見るだけで事業の全体像を瞬時に把握できる。
-   **下段：チャートエリア**
    -   **役割**: より詳細な分析を行うためのグラフやビジュアライゼーションを表示する。このエリアは、ユーザーが自由にカスタマイズできることを特徴とする。

#### **チャートエリアのカスタマイズ機能**

ユーザーは、チャートエリアの表示を以下の通りにカスタマイズできる。

-   **レイアウト選択機能**: チャートエリアを「1分割」「2分割」「4分割」の中から選択できるスイッチを設ける。
-   **チャート選択機能**: 分割された各エリア（スロット）に、どのチャートを表示するかをユーザー自身が選択できる。
    -   各スロットに「チャートを追加」ボタンを配置し、クリックすると表示可能なチャートの一覧（例：「売上推移グラフ」「費用構成円グラフ」など）がモーダルで表示され、選択できる。
