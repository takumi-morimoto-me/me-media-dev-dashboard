# ADR 0001: テーブル列のテキスト折り返し問題の解決

- **ステータス**: 決定 (Decided)
- **日付**: 2025-10-07

## コンテキスト (Context)

月次予実管理テーブルを実装する過程で、列のヘッダー（「予算」「実績」「達成率」など）のテキストが、セルの幅が狭いために意図せず縦書きのように折り返される問題が発生した。

当初、Tailwind CSSの固定幅クラス (`w-[120px]`) を指定することでこの問題の解決を試みたが、これだけではテキストの折り返しを完全に防ぐことができなかった。

## 決定 (Decision)

テーブルのヘッダーセル (`TableHead`) およびデータセル (`TableCell`) に、以下の3つのTailwind CSSクラスを組み合わせて適用することで、この問題を解決することに決定した。

1.  `min-w-[{適切な幅}]`: 列の最小幅を保証する。（例: `min-w-[80px]`）
2.  `whitespace-nowrap`: テキストの折り返しを強制的に禁止する。
3.  `px-{余白}`: セル内の左右に余白（パディング）を追加し、視認性を向上させる。（例: `px-4`）

## 分析と学び (Analysis and Learning)

`width`プロパティだけでは、ブラウザのテーブルレイアウト計算アルゴリズムや、他のスタイルとの競合により、テキストの折り返しを完全に防げない場合がある。

テキストが絶対に折り返されないことを保証するには、`whitespace-nowrap`クラスの使用が最も確実かつ直接的な解決策である。

今後のテーブル実装においては、意図しないテキストの折り返しを防ぐため、`width`や`min-width`による幅の確保と合わせて、`whitespace-nowrap`を積極的に活用する。
